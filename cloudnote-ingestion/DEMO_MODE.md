# CloudNote Presentation DEMO MODE Guide

This guide describes how to configure, run, and present CloudNote using the local deterministic `DEMO_MODE` configuration.

---

## 1. Setup & Launching

Make sure `DEMO_MODE=true` is set in your `.env` file at the project root. This ensures the backend completely bypasses Redis server requirements and stops background workers from attempting real classroom connections.

### Local Development Startup

1. **Start the Backend API**:
   ```bash
   python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
   ```

2. **Start the React Frontend**:
   ```bash
   cd frontend
   npm run dev
   ```

### Docker Compose Startup (Alternative)

If presenting from Docker containers:
1. Rebuild and launch the services:
   ```bash
   docker compose down
   docker compose up --build -d
   ```
2. The frontend is accessible at: `http://localhost:3000`
3. The backend API is accessible at: `http://localhost:8000`

---

## 2. Resetting and Simulating States

You can instantly switch the active dashboard demonstration using the CLI helper. It updates all timestamps dynamically to the current calendar date.

### Simulate Successful Connection

To demonstrate a live classroom monitoring session with a successful connection and proof:
```bash
python reset_demo_state.py --state connected
```
- **Dashboard Effect**: Renders a active `CONNECTED` indicator, shows the current class card for "Operating Systems", and embeds the stable virtual classroom attendance proof.

### Simulate Connection Failure

To demonstrate the automated watchdog detecting a connection failure:
```bash
python reset_demo_state.py --state failed
```
- **Dashboard Effect**: Renders a red `FAILED` indicator, shows the active class card for "Design & Analysis of Algorithms", and embeds the stable connection lost warning alert image.

### Reset to Standby/Idle

To reset the scheduler to standby mode showing no active classes:
```bash
python reset_demo_state.py --state idle
```
- **Dashboard Effect**: Renders a purple `Scheduler Active` indicator showing that the system is ready to join the next class.

## 3. Core Presentation Features

When presenting to teachers or evaluators, showcase these key features:

1. **Today's Class Schedule**: Shows the timetable loaded deterministically from the local JSON cache.
2. **Current Active Class**: A prominent card at the top displaying the live session details, including subject, instructor, and join timestamp.
3. **Attendance Proof Screen**: Displays the mock screenshot proof showing that the attendance was logged.
4. **Modal Previews**: Click on any screenshot thumbnail under "Previously Joined Classes" or "Current Active Class" to trigger the smooth modal screenshot preview.
5. **Lecture Intelligence Summaries**: In the dashboard sidebar, you can review dynamic AI summaries generated by the platform for both completed lectures.
